# CLAUDE.md — Raven (Trellis fork)

This is Trellis's fork of Raven. Treat these rules as mandatory for all work
(human or AI) in this repository.

## Remotes
- `origin`   → https://github.com/Trellis-Ph/raven  (OUR FORK — source of truth)
- `upstream` → https://github.com/The-Commit-Company/raven  (upstream Raven)

`origin/main` is our integration branch and the single source of truth for what
runs in every environment. `upstream/develop` is pulled in only via a dedicated
branch + PR, never straight into `main`.

## Branch protection — main is sacred
- NEVER commit, push, rebase, or force-push directly to `main`.
- `main` only ever changes through a reviewed, merged Pull Request.
- No exceptions for "small" or "urgent" fixes — they still go through a branch + PR.

## Workflow for every change
1. Branch off the latest `main`:
   `git fetch origin && git switch main && git pull && git switch -c <type>/<short-desc>`
   (types: `feat/`, `fix/`, `chore/`, `docs/`)
2. Commit your work on that branch and push it to `origin`.
3. Open a PR into `main` on the fork. Fill in what changed and why.
4. Get it reviewed, then merge via the PR (squash preferred).
5. If the change touches the frontend, rebuild and commit the updated
   `raven/www/raven.html` pointer on your branch (see Build artifacts).

## Branch hygiene
- Delete each branch (local + remote) immediately after its PR merges.
- Keep the branch list minimal; stale branches are deleted on sight.
- Before starting work, prune: `git fetch --prune`.

## Environment sync — every environment tracks main
Prod and sandbox MUST always run exactly `origin/main`. Never run an environment
off a feature branch or a detached commit.

Deploy / sync procedure (Frappe bench, manual) on each server:

    cd ~/frappe-bench/apps/raven
    git fetch origin
    git checkout main
    git pull --ff-only origin main
    cd ~/frappe-bench
    bench build --app raven
    bench --site <site> migrate
    bench restart

Verify an environment is in sync (the two hashes must be equal):

    git -C ~/frappe-bench/apps/raven rev-parse HEAD
    git -C ~/frappe-bench/apps/raven rev-parse origin/main

If they differ, the environment is OUT OF SYNC — sync before doing anything else.

## Build artifacts
- Built bundles (`raven/public/raven/assets/index-*.js`) are gitignored and
  regenerated by `bench build`. Vite hashes are content-deterministic, so a clean
  build of a given commit reproduces the same hash.
- The committed `raven/www/raven.html` pins the current bundle hash. After any
  frontend change: `bench build --app raven`, then commit the updated
  `raven/www/raven.html` on your branch as `chore(build): ...`.

## Syncing upstream Raven
- `git fetch upstream`, branch from `main`, merge/cherry-pick the upstream changes
  there, open a PR into `main`. Never merge `upstream/*` straight into `main`.
